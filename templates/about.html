About

HANSEL is a companion to GRETIL, facilitating open exchange of academic-level Sanskrit e-texts, with a focus on diplomatic transcriptions of printed editions produced using OCR.

FAQ

How do I contribute? Just send an email to hanselrepository at Gmail, wherever you're at in your process. The “team” will help you from there.

Will HANSEL incorporate all of GRETIL? No, at least not all at once. HANSEL's focus is on new or newly modified material. GRETIL will have no problem remaining available online in the form of its official archive and, when the main site eventually goes down, mirrors.

How does HANSEL compare to GRETIL or SARIT? Like SARIT, HANSEL has an explicit data model, everything is versioned, and consistent metadata is required. Like GRETIL, HANSEL is all-purpose and easy to contribute to. Unlike either, the base format is plain-text, from which are produced XML and HTML.

How do we know HANSEL will survive? This project is independent, open-source, well documented, and built on standard and inexpensive technologies.

Name and logo

"Human-accessible" means that material will always exist in forms that the greatest number of students and scholars can use directly, such as plain-text and HTML.

"NLP-ready" means that the data will also be structured in anticipation of computational applications.

The haṃsa (swan, or bar-headed goose) symbolizes not only grace but also discernment (viveka). A world away, in the Grimms' tale, Hänsel and Gretel succeed against the odds by working together.

Under the Hood

HANSEL consists of two coordinated Git repositories on GitHub: hansel-data for e-texts and metadata, and hansel-app for the web app. Using Git takes care of fine-grained versioning. Public GitHub repos enable open-source access to everything, including PR-based change history.

The web app is written in Python and Javascript and runs on a Digital Ocean server which holds an up-to-date clone of the data repo. The data repo also contains some Python and Bash automation code.
