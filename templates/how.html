How It Works

HANSEL's central data format is TEI-XML, but users and material contributors need not interact with it. Instead, users can access plain-text and HTML transforms, also as cumulative downloads, and material contributors can simply email in whatever format they have. Plain-text is the main processing format, and a simple plain-text markup scheme helps guide work on ensuring both essential structural elements and machine-actionability.

The below design diagram shows HANSEL's multiple community entry points, data formats, and use of automation.

In addition to automated checks on the body of draft material, there are also automated checks on metadata. Items must pass all validations in order to move from draft status to full publication, but acceptably structured draft material is also presented on the website, marked as such. These checks, part of the code base, are run locally by technical team members and also automatically on GitHub.

Finally, the website is dynamically served by a web application, which also takes care of metadata search and basic transliteration options.
